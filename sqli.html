📝 Soal Pemahaman SQL Injection

Jawab 3 soal di bawah ini untuk menguji pemahamanmu setelah praktik!

Soal 1 — Kenapa kamu bisa login tanpa password?

Kamu berhasil login padahal password salah. Kenapa ini bisa terjadi?

Petunjuk: Perhatikan bagaimana input ' OR '1'='1 mengubah logika query.
A. Websitenya error
B. Input ' OR '1'='1 bikin '1'='1' selalu BENAR
C. Admin udah ada di database
D. Sistem ngasih akses ke semua

Soal 2 — Apa bedanya query normal vs payload?

Bandingkan kedua query di bawah ini!

Petunjuk: Bandingkan kondisi WHERE pada kedua query — apa yang membuat query payload selalu benar?
Normal: SELECT * FROM users WHERE username='admin' AND password='12345'
Payload: SELECT * FROM users WHERE username='admin' OR '1'='1' AND password='12345'
A. Query normal lebih cepat
B. Query normal butuh password bener, payload bisa lolos
C. Pake database beda
D. Query normal cuma nyari admin

Soal 3 — Cara paling ampuh cegah SQL Injection?

Dari yang udah kamu pelajari, mana cara pencegahan yang PALING TEPAT?

Petunjuk: Cara paling ampuh adalah yang membuat input dianggap DATA, bukan PERINTAH SQL.
A. Bikin password panjang
B. Parameterized Queries — input dianggap DATA
C. Ganti nama tabel
D. Batasi jumlah user

📊 Analisis SQL Injection

Cara Masuk
Menyisipkan query SQL ke input yang tidak divalidasi
Cara Menyebar
Tidak menyebar sendiri — butuh eksekusi query
Dampak Utama
Bypass login, pencurian data, modifikasi database
Cara Deteksi
Pantau log, WAF, penetration testing
🛡️ Pencegahan
Parameterized Queries, validasi input, batasi hak akses, WAF
