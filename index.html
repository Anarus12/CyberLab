📊 Analisis Serangan SQL Injection

Cara Masuk ke Sistem
Menyisipkan query SQL berbahaya ke input form (username/password) yang tidak divalidasi
Cara Menyebar
Tidak menyebar sendiri — membutuhkan eksekusi query melalui input form yang rentan
Dampak Utama
Bypass autentikasi, pencurian data, modifikasi database, hingga eksekusi perintah sistem
Cara Deteksi
Pantau log error, gunakan WAF (Web Application Firewall), lakukan penetration testing
🛡️ Cara Pencegahan
Parameterized Queries, validasi input, batasi hak akses database, enkripsi data sensitif, aktifkan WAF

📝 Soal Pemahaman SQL Injection

Jawab 3 soal di bawah ini untuk menguji pemahamanmu setelah praktik!

Soal 1 — Kenapa kamu bisa login tanpa password?

Kamu berhasil login padahal password salah. Kenapa ini bisa terjadi?

A. Karena websitenya lagi error
B. Karena input ' OR '1'='1 bikin kondisi '1'='1' selalu BENAR
C. Karena username admin emang udah ada di database
D. Karena sistemnya ngasih akses ke semua orang

Soal 2 — Apa bedanya query normal sama query yang disisipi payload?

Query normal: SELECT * FROM users WHERE username='admin' AND password='12345'
Query payload: SELECT * FROM users WHERE username='admin' OR '1'='1' AND password='12345'

A. Query normal lebih cepat jalannya
B. Query normal butuh password bener, query payload bisa lolos tanpa password
C. Query normal pake database beda
D. Query normal cuma nyari admin, query payload nyari semua data

Soal 3 — Cara paling ampuh cegah SQL Injection?

Dari yang udah kamu pelajari, mana cara pencegahan yang PALING TEPAT?

A. Bikin password yang panjang dan rumit
B. Pake Parameterized Queries — input dianggap DATA, bukan PERINTAH SQL
C. Ganti-ganti nama tabel biar hacker bingung
D. Batasi jumlah user yang bisa login
02 / MALWARE PENYAMAR

Simulasi Trojan

Trojan menyamar sebagai file yang terlihat aman. Begitu dijalankan, file tersebut diam-diam mengirim data korban ke penyerang. Klik file di bawah untuk mensimulasikannya.

📄
Undangan_Ulang_Tahun.pdf.exe
Mengirim data ke server penyerang...0%
File yang tampak seperti dokumen biasa ternyata adalah program tersembunyi. Saat dibuka, ia langsung menjalankan aksinya di latar belakang tanpa sepengetahuan korban — itulah sifat khas Trojan: menyamar sebagai sesuatu yang dipercaya.

📊 Analisis Trojan

Cara Masuk ke Sistem
Disamarkan sebagai file/software legal yang diunduh korban (game, tool, PDF, dll)
Cara Menyebar
Tidak menyebar sendiri — butuh aksi pengguna (download & eksekusi)
Dampak Utama
Pencurian data (password, foto, dokumen), membuka backdoor, menonaktifkan keamanan
Cara Deteksi
Antivirus/antimalware, perilaku mencurigakan (koneksi keluar yang tidak dikenal), analisis file
🛡️ Cara Pencegahan
Unduh dari sumber resmi, periksa ekstensi file (.exe/.apk vs .pdf/.doc), aktifkan antivirus, jangan klik link sembarangan

Simulasi Worm

📊 Analisis Worm

Cara Masuk ke Sistem
Eksploitasi celah keamanan pada sistem operasi atau aplikasi jaringan (tanpa interaksi pengguna)
Cara Menyebar
❌ Menyebar otomatis melalui jaringan, USB, email — mereplikasi diri sendiri
Dampak Utama
Konsumsi bandwidth, melumpuhkan jaringan, membawa payload berbahaya (ransomware, backdoor)
Cara Deteksi
Monitor lalu lintas jaringan (traffic anomali), antivirus, intrusion detection system (IDS)
🛡️ Cara Pencegahan
Update sistem operasi, nonaktifkan autorun USB, gunakan firewall, segregasi jaringan

📊 Analisis Backdoor

Cara Masuk ke Sistem
Ditanam oleh malware lain (Trojan) atau dieksploitasi dari celah keamanan yang sudah ada
Cara Menyebar
Tidak menyebar — diinstal sekali, memberi akses permanen ke penyerang
Dampak Utama
Kontrol jarak jauh, pencurian data, pemasangan malware tambahan, pemanfaatan untuk serangan lain
Cara Deteksi
Audit log akses, scan port terbuka, analisis proses mencurigakan, intrusion detection system (IDS)
🛡️ Cara Pencegahan
Ganti password default, tutup port tidak digunakan, multi-factor authentication (MFA), audit log berkala
05 / PRAKTIK JARINGAN

Praktik Susunan Kabel UTP

Pilih jenis kabel, pelajari urutan warna, lalu verifikasi susunanmu. Jika salah, lihat analisa dampaknya.

🔧 6 Langkah Crimping Kabel UTP

Praktik langsung dengan kabel, konektor RJ45, dan tang crimping:

  1. Kupas kulit luar kabel ±2 cm
  2. Urutkan warna sesuai standar (lihat referensi di bawah)
  3. Potong rata ujung kabel
  4. Masukkan ke konektor RJ45 (pastikan urutan tidak berubah)
  5. Crimping dengan tang (tekan kuat sampai klik)
  6. Tes dengan LAN tester atau colok ke PC
Referensi Urutan Warna:
Pin 1: Putih-Oranye | Pin 2: Oranye | Pin 3: Putih-Hijau | Pin 4: Biru | Pin 5: Putih-Biru | Pin 6: Hijau | Pin 7: Putih-Coklat | Pin 8: Coklat
✅ Kedua ujung sama → STRAIGHT (T568B - T568B)

⚠️ Dampak Kesalahan Urutan Warna

  • Koneksi Gagal — Data tidak sampai ke tujuan
  • CRC Error — Paket data rusak karena TX/RX tidak sinkron
  • Crosstalk & Interferensi — Pasangan kabel tidak sesuai standar
  • Kerusakan Perangkat — Pada kasus ekstrem, perangkat bisa rusak
  • Waktu Pengerjaan — Harus mengulang crimping dari awal
